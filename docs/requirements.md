# Requirements

## 1. 项目目标

本项目使用 AVS Lab Basilisk 搭建一个面向航天器姿态控制的仿真工程骨架。第一阶段以官方 `scenario_AttGuidance` 为基线，建立一个可扩展的 6-DOF 姿轨一体仿真框架，并在此基础上逐步替换为自定义任务模式、传感器和执行机构配置。

## 2. 任务模式

- `standby`：待机模式，不主动输出姿态控制命令，仅保持系统运行和状态记录。
- `hillPoint`：基线模式，使航天器姿态相对于轨道局部参考系收敛，用于验证主控制链路。
- `inertialPoint`：惯性定向模式，对准固定惯性方向，用于验证非轨道相关姿态控制。
- `sunSafePoint`：太阳安全模式，失效保护时优先保证供电与热安全。
- `slewTransition`：模式切换过程中的姿态机动模式，用于不同指向模式之间的平滑过渡。
- `wheelDesaturation`：飞轮卸载模式，作为第二阶段扩展项，解决长期运行中的动量积累问题。

## 3. 传感器需求

- 基线传感器：`SimpleNav`，提供理想姿态与轨道导航信息，用于第一阶段闭环验证。
- 轨道与天体信息：基于 SPICE 与重力体工厂提供轨道参考和天体状态。
- 第二阶段预留：
  - 星敏感器，用于高精度姿态估计；
  - 陀螺/IMU，用于角速度测量；
  - 粗太阳敏感器，用于太阳安全模式；
  - 轨道导航输入，用于更真实的轨道估计链路。

## 4. 执行机构需求

- 主执行机构：4 个 reaction wheel pyramid，作为主要姿态控制执行机构。
- 基线控制输出：机体控制力矩经 `rwMotorTorque` 分配为飞轮电机力矩命令。
- 第二阶段预留：
  - 磁力矩器，用于飞轮卸载；
  - ACS 推力器，用于故障模式或轨控扩展；
  - 外部扰动力矩接口，用于鲁棒性测试。

## 5. 验收指标

- 功能验收：
  - 官方基线 `scenario_AttGuidance` 可在本机稳定运行；
  - 自定义工程仓库结构完整，可承载场景、配置、测试与结果输出；
  - 至少实现 `standby`、`hillPoint`、`inertialPoint` 三种模式的接口预留。
- 性能验收：
  - 在基线 `hillPoint` 仿真中，10 分钟仿真内无崩溃、无数值发散；
  - 姿态误差在 120 秒内收敛到可接受范围；
  - 稳态姿态误差不大于 `0.5 deg`；
  - 稳态角速度误差不大于 `0.05 deg/s`；
  - 基线场景中飞轮不出现持续饱和。
- 输出验收：
  - 必须能输出姿态误差、角速度误差、控制力矩曲线；
  - 必须保留模式切换点与关键参数配置记录；
  - 结果文件应能够支持后续撰写实验报告或阶段总结。

## 6. 当前阶段边界

- 当前阶段重点是“姿态控制工程骨架”，不是完整在轨任务系统。
- 高保真传感器噪声、执行机构故障注入、轨控与编队飞行逻辑均放在后续阶段扩展。
